# 编辑文本同步到生视频的 video_prompt

## 修改文件

`[/Users/bytedance/Downloads/## 动作.ini](/Users/bytedance/Downloads/## 动作.ini)`

## 修改内容

### 1. 更新"编辑文本处理"规则（第73-83行）

当前逻辑：

```
if 是否修改文本 == false:
    所有 prompt = ""
    video_prompt = ""
else:
    分析用户文本，将图像编辑需求和视频动态需求统一填入 video_prompt
```

修改为：

```
if 是否修改文本 == false:
    所有 prompt = ""
    video_prompt = ""
else:
    分析用户文本：
    - 视频动态需求 → 填入 video_prompt
    - 图像编辑需求 → 填入对应生图工具的 prompt
    - 如果生图工具的 prompt 非空，将其内容也追加到 video_prompt
    - 最终 video_prompt = 视频动态需求 + 生图编辑需求（拼接）
```

同时更新注意说明（第83行），原文 "AI_movie_CY工具的编辑需求统一放video_prompt，不区分图像编辑和视频动态" 需修改为体现新逻辑：图像编辑需求同时放入生图 prompt 和 video_prompt。

### 2. 更新示例2（第192-223行）

当前：`video_prompt = "把衣服变成红色，表情更丰富一点"`，ID2i `prompt = ""`

- "把衣服变成红色"属于图像编辑需求 → 应填入 ID2i 的 prompt
- "表情更丰富一点"属于视频动态需求 → 填入 video_prompt
- ID2i prompt 非空，其内容追加到 video_prompt
- 最终：ID2i prompt = "把衣服变成红色"，video_prompt = "表情更丰富一点，把衣服变成红色"

### 3. 更新示例7（第314-345行）

- AI_create prompt 非空 → 追加到 video_prompt
- 最终：video_prompt = "使用{user_image1}的脸和{user_image2}的衣服"
