# 可编辑子图 + 自由连线

## 问题

当前子 Agent 是通过 Agent-as-a-Tool 模式调用的 — 父 Agent 的 LLM 决定调用顺序。
但用户想要:
1. 子 Agent 之间有**显式的数据流** (image_agent 输出 → video_agent 输入)
2. **拖拽连线**建立这种关系，不需要写 System Prompt
3. 实线 = 必经路径，虚线 = 条件路径 (如"有图片素材时才经过 image_agent")

## 设计

### 1. agent_edges 表

```python
class AgentEdge(Base):
    __tablename__ = "agent_edges"

    id = Column(Integer, primary_key=True, autoincrement=True)
    parent_id = Column(String(64), nullable=False, index=True)  # 所属父 Agent
    source_id = Column(String(64), nullable=False)              # 子 Agent 起点
    target_id = Column(String(64), nullable=False)              # 子 Agent 终点
    edge_type = Column(String(16), default="solid")             # solid | dashed
    condition = Column(Text, default="")                        # 条件表达式 (dashed 时)
```

示例:
```
parent_id: "producer"
source_id: "image_agent"   target_id: "video_agent"   edge_type: "dashed"   condition: "uploaded_assets 包含 image"
```

### 2. GenericAgent 子图执行模式

当父 Agent 的 available_tools 全是子 Agent (无直接 Skill) 且存在 agent_edges 时，
不再走 ReAct 工具调用，改为**按拓扑序执行子图**:

```python
async def _execute_subgraph(self, config, state, project_id):
    children = await get_child_agents(config.id)
    edges = await get_agent_edges(config.id)

    # 构建拓扑序
    order = topological_sort(children, edges)

    # 逐步执行
    accumulated_output = {}
    for child_id in order:
        child_config = next(c for c in children if c.id == child_id)

        # 检查入边条件 (dashed 边可能被跳过)
        if child_config.bypass or not self._check_edge_conditions(child_id, edges, state):
            continue

        # 合并上游输出到 state
        child_state = {**state, **accumulated_output}
        result = await GenericAgent().run(child_config, child_state, project_id)
        accumulated_output.update(result)

    return accumulated_output
```

判断逻辑:
- 有子 Agent + 有 edges → 子图执行模式
- 有子 Agent + 无 edges → ReAct 工具调用模式 (当前行为)
- 无子 Agent → 直接用 Skill 执行

### 3. 边条件引擎

复用 SkillRegistry 的 trigger 匹配逻辑:
```python
def _check_edge_conditions(self, target_id, edges, state):
    incoming = [e for e in edges if e.target_id == target_id]
    for edge in incoming:
        if edge.edge_type == "solid":
            return True  # 实线 = 必经
        if edge.edge_type == "dashed" and edge.condition:
            # 复用 trigger 匹配
            return self._eval_trigger(edge.condition, state)
    return True  # 无入边 = 起始节点，始终执行
```

### 4. API

```
GET    /api/admin/agents/{parent_id}/edges       — 获取子图边列表
POST   /api/admin/agents/{parent_id}/edges       — 创建边 (拖拽连线)
DELETE /api/admin/agents/{parent_id}/edges/{id}   — 删除边
PUT    /api/admin/agents/{parent_id}/edges/{id}   — 更新边 (改条件/类型)
```

### 5. React Flow 编辑模式

子 Agent 视图 (点击 Producer 后看到的) 开启 React Flow 编辑:

```typescript
<ReactFlow
  nodes={subAgentNodes}
  edges={subAgentEdges}
  onConnect={(params) => createEdge(params)}  // 拖拽连线 → POST API
  onEdgeClick={(_, edge) => deleteEdge(edge)} // 点击边 → DELETE API
  nodesDraggable={true}
  nodesConnectable={true}  // 允许拖拽连线
>
```

边的样式:
- 实线: `stroke: #22c55e, strokeWidth: 2` (绿色，必经)
- 虚线: `stroke: #eab308, strokeWidth: 2, strokeDasharray: "6 3"` (黄色，条件)
- 条件文字标注在边上

### 6. 用户操作流程

Producer 拆分为 image_agent + video_agent 的完整操作:

1. 点击 Producer 节点 → [+ 子Agent] → 创建 image_agent
2. [+ 子Agent] → 创建 video_agent
3. 给 image_agent 分配 Skill: generate_video_i2v
4. 给 video_agent 分配 Skill: generate_video_t2v, generate_video_r2v, generate_video_extend
5. 在画布上从 image_agent 拖一条线到 video_agent
6. 右键这条线 → 设为虚线 → 配置条件: "uploaded_assets 包含 image"

结果: 
- 有图片素材 → image_agent 执行 → 输出传给 video_agent → video_agent 执行
- 无图片素材 → 跳过 image_agent → video_agent 直接执行
