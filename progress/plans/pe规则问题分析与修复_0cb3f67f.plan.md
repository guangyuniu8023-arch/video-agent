# PE 规则问题分析与修复方案

## 问题背景

用户输入为**口播剧本**（第一人称独白 + 场景描述 + 镜头指示混合），但 PE 的规则和 Few-shot 示例全部针对**第三人称场景描述**，导致 LLM 在处理此类输入时行为不可预期。

---

## 问题 1：缺少"你"的排除规则（高优先级）

**现状**：PE 第75行列举了需要替换的代词："我"、"她"、"他"，但完全没有提及"你"。

**风险**：LLM 可能将"你看我的讲了这么久嘴都干了"中的"你"也替换为 `{role_0}`，导致输出变成 `{role_0}看{role_0}的讲了...`，语义完全错误。

**修复建议**：在核心规则第3条中明确添加排除说明：

```
注意："你"、"你们"等面向观众/听众的代词不做替换，这类代词指向的是视频观众而非画面中的角色。
```

---

## 问题 2：缺少口播/第一人称独白类 Few-shot 示例（高优先级）

**现状**：全部14个 Few-shot 示例都是**第三人称场景描述**格式（"让图1穿..."、"一个男生在..."），没有任何一个示例涉及**第一人称口播/独白**格式。

**风险**：口播文本的特点是：

- "我"是说话者的自称（不同于场景描述中"我"≈"他"的混用）
- 文本中存在面向观众的"你"
- 包含口语化的语气词和互动用语

LLM 没有相关示例参考，处理行为不稳定，可能：

- 不替换"我"（认为这是说话者的自然自称，不该替换）
- 错误替换"你"
- 把口播内容改写成第三人称

**修复建议**：添加 1-2 个口播类示例，例如：

```
### 示例15：单图 + 口播/第一人称独白

**输入**：
图片1: [AI形象]
用户文本: 一个女生博主在镜头前说：大家好，我今天给你们推荐一个好东西，
我用了之后真的觉得太好了！你们赶紧去试试吧！

**推理过程（不输出）**：单图无显式引用。"一个女生博主"是人物描述词→替换为
{role_0}。"我"出现2次，均指代说话者本人（即图中角色）→替换为{role_0}。
"你们"是面向观众的称呼→不替换。

**输出**：{role_0}在镜头前说：大家好，{role_0}今天给你们推荐一个好东西，
{role_0}用了之后真的觉得太好了！你们赶紧去试试吧！
```

---

## 问题 3：缺少镜头指示/导演备注的处理规则（中优先级）

**现状**：用户文本中包含 `（推镜头到嘴部看特写，嘴部有一点干）`，这是**镜头/导演指示**，不是角色台词。PE 中没有任何关于此类内容的处理规则。

**风险**：LLM 不知道该：

- 保留原样（可能正确，取决于下游系统是否识别）
- 删除掉
- 尝试在其中也做槽位替换

**修复建议**：在注意事项中添加：

```
- **导演指示/镜头语言**：括号内的导演指示如"（推镜头到...）"、
 "（特写）"等内容保留原样，不做槽位替换，这类内容是给生成系统的指令而非角色行为描述
```

---

## 问题 4：人物描述词的边界不清晰（中优先级）

**现状**：PE 第75行举例 "一个女生"、"男孩"，但实际输入是 "一个女生博主"。

**风险**：LLM 可能：

- 只替换 "一个女生" → 输出 `{role_0}博主口播`（"博主"残留，不自然）
- 替换整个 "一个女生博主" → 输出 `{role_0}口播`（正确）

**修复建议**：在规则中明确说明人物描述词应**整体替换**：

```
人物描述词指完整的人物称谓短语（如"一个女生博主"、"穿红衣服的男孩"、
"旁边的大叔"），替换时应将整个短语替换为 {role_N}，而非只替换其中一部分。
```

---

## 问题 5：缺少"元描述 + 实际内容"混合格式的处理（低优先级）

**现状**：用户输入是 `一个女生博主口播，情绪比较激动，口播内容：[实际口播台词]` 这种"场景描述 + 台词"的混合格式。PE 没有覆盖此场景。

**风险**：LLM 可能只处理前半部分的场景描述，或者把后半部分当成独立文本处理，导致前后不一致。

**修复建议**：在 Few-shot 中加入包含场景描述 + 台词内容的复合示例（可以和问题2的示例合并）。

---

## 修改总结

按优先级排序的修改清单：


| 优先级 | 修改项                     | 修改位置                    |
| --- | ----------------------- | ----------------------- |
| 高   | 添加"你/你们"排除规则            | 核心规则 - 第3条隐式引用补全        |
| 高   | 添加口播/第一人称独白 Few-shot 示例 | Few-shot 示例区（建议示例15、16） |
| 中   | 添加镜头指示/导演备注处理规则         | 注意事项                    |
| 中   | 明确人物描述词的整体替换边界          | 核心规则 - 第3条              |
| 低   | 添加混合格式（元描述+台词）示例        | Few-shot 示例区            |
