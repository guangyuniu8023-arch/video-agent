# 优化视频提示词安全重构 Prompt

目标文件：`/Users/bytedance/Downloads/##Role: \n你是一位专业的短视频导演兼高级视觉分析师，擅长将动态舞蹈视频.ini`

## 第一步：修复 JSON 语法错误

当前文件有大量 JSON 语法问题，会导致解析失败。需要修复：

- `style_intent_extraction` 数组（第 14-16 行）：元素之间补逗号
- `full_decoupling` 数组（第 58-60 行）：元素之间补逗号
- `style_definition`（第 70-77 行）：将 `{...]`  改为合法的数组 `[...]`
- `prohibited_elements` 数组（第 83-84 行）：元素之间补逗号
- `background_reconstruction` 数组（第 97 行）：元素之间补逗号
- `output_requirements` 数组（第 119 行）：数组闭合后补逗号
- `user_editing_adaptation` 与 `outfit_reconstruction` 之间（第 52-53 行）：补逗号

## 第二步：放宽材质/色系约束，扩大有效输出空间

当前 `prohibited_elements` 禁止了醋酸、缎面、真丝、雪纺、亮面反光等材质，配合低饱和色系要求，可选空间极窄。优化方向：

- 将"全面禁止"改为"分场景限制"：例如在力量型/先锋风格下允许皮革光泽，在少女/优雅风格下允许有节制的丝质
- 将颜色约束从"严禁复用原色 + 必须低饱和"放宽为"严禁复用原色，鼓励在对应风格色盘内选择"，为每个风格定义独立的推荐色盘

## 第三步：建立服装风格与场景的绑定映射

在 `style_definition` 和 `scene_style_keywords` 之间建立明确的适配关系，避免模型随机搭配导致不协调：

- 理性极简/建筑感 --> 先锋/秩序感场景、极简/自然原野
- 冷感松弛/北欧 --> 慵懒/法式流、日韩街道
- 先锋比例/强态度 --> 先锋/秩序感、野性流
- 少女/知性优雅 --> 慵懒/法式流、日韩街道、暗调/奢靡风

## 第四步：补充多样化示例（关键）

当前仅有 1 个示例（夹克+直筒裤+电梯厅），需补充至少 3 个覆盖不同风格的示例：

- 示例 2（少女/知性风格）：连衣裙或短上衣+裙装搭配 + 法式阳台/咖啡厅场景
- 示例 3（冷感松弛/北欧风格）：宽松针织+阔腿裤叠穿 + 日韩街道/居民区场景
- 示例 4（先锋/力量型风格）：廓形大衣或夸张比例单品 + 废弃天台/水泥建筑场景

每个示例需确保服装的版型、领口、色系、面料四个维度互不重复。

## 第五步：增加"风格轮盘"机制替代不可执行的"禁止连续输出同类风格"

将第 72 行的 `"随机调用：禁止连续输出同类风格"` 替换为可执行的工程化方案：

- 在 prompt 中新增一个 `style_wheel` 输入槽位，格式为：`"上次使用风格：xxx"`
- 增加规则："若提供了上次使用风格，则本次必须从不同风格大类中选取"
- 若未提供该字段，则模型自行从风格库中选取

## 第六步：优化 Anti-Duplication 规则的可执行性

当前"四维度全部不能重合"过于严格。优化为：

- 将"四维度必须全部漂移"改为"至少三个维度必须漂移"
- 增加一条"视觉第一印象测试"作为最终校验：重构后的服装整体轮廓（oversized/fitted/layered）必须与原服装不同
