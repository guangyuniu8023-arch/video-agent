# 收紧表演类诉求的"匹配场景"定义

## 问题

当前约束条件4（第71行）写的是：

> 唱歌/rap/相声/快板/跳水/舞台舞蹈等：隐含了与之匹配的专业场景氛围，除非原图场景本身就是匹配场景（如舞台），否则 → r2v

存在两个问题：

- "脱口秀"没有被显式列入表演类别，模型可能将其归入"口播"类而走 i2v
- "匹配场景（如舞台）"的定义过于宽泛，模型可能将酒吧、餐厅等偶尔举办演出的场所也视为"匹配场景"

## 修改方案（2处改动）

### 改动1：修改约束条件4的表演类规则（第71行）

**原文**：

```
- 唱歌/rap/相声/快板/跳水/舞台舞蹈等：隐含了与之匹配的专业场景氛围，除非原图场景本身就是匹配场景（如舞台），否则 → r2v
```

**改为**：

```
- 唱歌/rap/脱口秀/相声/快板/跳水/舞台舞蹈等：隐含了与之匹配的专业场景氛围，除非原图中明确可见专业表演设施（如舞台台面、聚光灯、话筒架、观众席等），否则 → r2v。注意：酒吧、餐厅、咖啡厅等场所即使偶尔举办演出，只要图中没有可见的专业表演设施，仍不视为匹配场景。
```

关键变化：

- 显式加入"脱口秀"
- 将"匹配场景"从抽象的"如舞台"改为具体的可视化判断标准：**原图中是否可见专业表演设施**
- 增加排除说明：酒吧/餐厅/咖啡厅等不自动算匹配

### 改动2：增加示例17（酒吧+脱口秀 → r2v）

在现有示例16之后添加：

```
### 示例17：表演类诉求，场所非专业表演场景 → r2v
图片：一个女生站在酒吧吧台前
指令：说一段脱口秀
输出：r2v
原因：脱口秀属于表演类诉求，需要匹配的专业表演场景氛围；虽然酒吧偶尔举办脱口秀，但图中没有可见的舞台、聚光灯、话筒架等专业表演设施，不视为匹配场景
```

## 影响评估

- 酒吧/餐厅/咖啡厅 + 脱口秀/唱歌/rap → 从可能误判i2v **修正为** r2v
- 舞台 + 脱口秀 → 仍然 i2v（舞台有专业表演设施）
- 口播类 → 不受影响（口播规则在前，优先匹配）
- 简单身体动作 → 不受影响
- 其他已有案例 → 不受影响（修改范围仅限表演类的场景匹配定义）
