# 画布模块化重构 v4

## 问题

1. 子画布 (ContainerDrilldownView) 是只读展示，不能双击添加、点击配置、拖拽连线
2. 父 Agent 有子 Agent 时，面板没有串行/并行切换按钮
3. 子画布里点击节点，右侧面板显示不正确

## 方案

### 1. 子画布复用主画布组件

不再用独立的 ContainerDrilldownView，改为让 AgentFlowCanvas 支持 `parent_canvas` 过滤：

- 主画布: 加载 parent_canvas=null 的节点
- 子画布: 加载 parent_canvas=containerId 的节点
- AgentFlowCanvas 的所有交互 (双击/点击/连线/NodePicker) 在子画布也可用

WorkspacePage 维护一个 `canvasStack` 导航栈:

- 主画布 → 双击容器 → push 子画布 → 双击容器 → push 更深层
- 面包屑显示栈路径，点击任一层级返回

### 2. 父 Agent 面板恢复执行模式按钮

AgentDetailView 检测当前 Agent 是否有子 Agent (parent_id 指向它的 agent)，有则显示串行/并行按钮。

### 3. 子画布节点的右侧面板

selectedNodeId 在主画布和子画布之间共享，NodeDetailPanel 根据 nodeId 前缀判断类型并正确渲染。
