# 修改剧本路由逻辑：动作仅限跳舞，剧本兜底

## 核心变更

修改文件：`[/Users/bytedance/Downloads/## 剧本](/Users/bytedance/Downloads/## 剧本)`

**原决策顺序**：反差 -> 唱歌 -> 剧本(ASR叙事) -> 动作(兜底)

**新决策顺序**：反差 -> 唱歌 -> 动作(明确跳舞) -> 剧本(兜底)

## 具体修改点

### 1. 修改"核心主旨识别"文字描述（第14-17行）

将原来的四条规则改为：

- 如果还原视频中的前后状态反差就能最大程度还原视频内容 -> 走反差
- 如果还原视频中角色的口型对唱就能最大程度还原视频内容 -> 走唱歌
- 如果视频内容为明确的跳舞（包括但不限于：编舞、手势舞、广场舞、街舞、宅舞、民族舞等所有类型的舞蹈）-> 走动作
- 以上都不命中 -> 走剧本（兜底）

### 2. 修改伪代码块（第26-41行）

将判断流程重写为：

```
Step 1: 判断反差
  if 前后有明显状态变化（变装/变妆/场景切换）:
      return 路由到"反差"

Step 2: 判断唱歌
  if ASR有人声 and ASR内容为歌词:
      if 画面中人物口型与ASR歌词内容对应:
          return 路由到"唱歌"

Step 3: 判断动作
  if 画面中人物在跳舞（所有类型的舞蹈，包括手势舞）:
      return 路由到"动作"

Step 4: 兜底剧本
  return 路由到"剧本"（使用AI_movie_CY）
```
