# 编辑文本过滤非编辑类词

## 修改范围

## 修改内容

在 `else` 分支的"分析用户文本"之前，增加一步过滤预处理。三个文件的逻辑块统一改为：

```
else:
    先过滤用户文本，去掉非编辑类描述词，只保留实际编辑指令：
    - 过滤掉：挑战描述（"xxx挑战"、"xxx翻跳"、"xxx舞蹈"）、同款描述（"同款xxx"）、纯标签/话题词等
    - 保留：实际的图像编辑指令和视频动态指令
    
    用过滤后的文本，拆分为两类需求：
    - 图像编辑需求 → 填入对应生图工具的 prompt
    - 视频动态需求 → 填入 video_prompt
    - 如果生图工具的 prompt 非空，将其内容也追加到 video_prompt
    - 最终 video_prompt = 视频动态需求 + 生图编辑需求（拼接）
```
