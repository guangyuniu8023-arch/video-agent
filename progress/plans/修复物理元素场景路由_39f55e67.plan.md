# 修复"物理元素存在但被误判"的路由识别问题

## 问题定位

当前 Prompt 的"常识判断"规则（约束条件2，第47行）侧重于**场景与行为的宏观联想关系**（如"舞台↔脱口秀"、"咖啡厅↔街舞"），但缺少对**微观物理元素匹配**的显式引导。导致模型在遇到"原图中存在指令所需的关键物理元素（如墙），但整体场景氛围不是典型关联场景"时，倾向于按宏观氛围不匹配而走 r2v。

现有的"白墙刷粉色"示例（第53行）虽然暗含了这个逻辑，但它：

- 嵌在"造型不影响判断"的说明文字中，不够突出
- 没有被抽象为一条通用规则
- 没有对应的正式 Example

## 修改方案（共3处改动）

### 改动1：在约束条件2中增加"物理元素匹配"子规则

在 `[Untitled-7](Untitled-7)` 第53行（"重要：常识判断仅针对..."段落）之后，增加一条新的子规则：

```
**补充：当文本指令中动作的直接作用对象（如墙、桌、门、窗、地面等具体物理元素）在原图中明确可见时，视为场景满足该动作的基本条件，优先走 i2v。** 判断重点是"动作能否在原图中发生"，而非"原图的整体氛围是否是该动作的典型场景"。例如：
- 图片（女生在室内沙发上自拍，背后有墙）+ 指令（在墙上画涂鸦）→ 墙在图中可见，动作可以在原图中发生 → i2v
- 图片（男生站在室内，身后有门）+ 指令（打开门走出去）→ 门在图中可见，动作可以从原图延续 → i2v

注意：此规则仅适用于简单的物理交互动作。如果文本在物理交互基础上还引入了原图不存在的新氛围/新元素（如"在墙上贴满符咒，妖气弥漫"），则仍需按核心判断逻辑评估是否走 r2v。
```

**为什么不会影响其他场景**：

- 这条规则有明确的触发条件——"动作的直接作用对象在原图中可见"
- 有明确的排除条件——"引入新氛围/新元素时仍按核心逻辑判断"
- 不改变现有任何规则的优先级，只是对"常识判断"的一个补充细化

### 改动2：增加1个正式示例

在现有示例14之后，添加示例15：

```
### 示例15：动作作用对象在原图中可见 → i2v
图片：一个女生在室内沙发上自拍，背后有墙
指令：我在墙上画涂鸦，一边画一边涂鸦
输出：i2v
原因：墙在原图中明确可见，"画涂鸦"是对墙的直接物理交互，动作可以在原图场景中发生，无需换场景
```

### 改动3（可选）：增加1个反面示例作对比

紧接示例15，添加示例16作为边界对照：

```
### 示例16：动作作用对象不在原图中 → r2v
图片：一个女生在室内沙发上自拍，背后有墙
指令：我在黑板上画涂鸦
输出：r2v
原因：原图中没有黑板，文本指定的动作对象不存在于原图场景中，需要重新构建场景
```
