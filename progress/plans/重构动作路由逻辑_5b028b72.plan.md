# 重构"动作"路由逻辑

## 修改文件

[## 角色定义](## 角色定义)

## 变更概要

将决策流程从 `反差 → 唱歌 → 剧本 → 动作(兜底)` 改为 `反差 → 唱歌 → 动作(条件匹配) → 剧本(条件匹配) → 剧本(兜底)`。

## 具体改动

### 1. 修改"核心主旨识别"段落（第12-41行）

**新的判断流程**：

```
Step 1: 判断反差（不变）

Step 2: 判断唱歌（不变）

Step 3: 判断动作（改为条件匹配，只看视频画面）
  if 画面中人物明确在跳舞或以肢体动作为视频主题:
      return 路由到"动作"

Step 4: 判断剧本（保留原条件，位置从Step 3移到Step 4）
  if ASR有人声 and ASR内容为叙事（对白/独白/旁白/故事讲述，不是歌词）:
      return 路由到"剧本"（使用AI_movie_CY）

Step 5: 兜底剧本
  return 路由到"剧本"（使用AI_movie_CY）
```

关键变化：

- **动作**：从兜底变为条件匹配，判断依据**只看视频画面**（VLM识别人物是否在跳舞/做明确肢体动作）
- **剧本**：保留ASR叙事性条件判断，同时作为最终兜底
- 动作判断位置提前到剧本之前

### 2. 更新受影响的示例

以下示例的路由结果会发生变化：

- **示例4（舞蹈动作）**：结果不变，仍走"动作"，但理由更新为"画面中人物明确在跳舞"
- **示例6（BGM氛围感摆拍）**：原来走"动作"(兜底) → 现在走"剧本"(兜底)，因为画面无明确舞蹈/肢体动作
- **示例8（创意透视挑战）**：原来走"动作"(兜底) → 现在走"剧本"(兜底)，因为画面无明确舞蹈/肢体动作
- **示例9（用户说剧本但实际是舞蹈BGM）**：原来走"动作"(兜底) → 需要根据画面判断，如果画面有明确肢体动作仍走"动作"，否则走"剧本"(兜底)

### 3. 新增/调整的示例场景

需要补充以下典型场景的示例，明确"动作"的正向命中条件：

- 明确舞蹈（画面有跳舞动作）→ 动作
- 明确肢体动作为主题（如手势舞、健身操）→ 动作
- 氛围感摆拍/静态展示（画面无明确舞蹈/肢体动作）→ 剧本(兜底)

### 4. 输出格式段落更新

- "路由到其他PE"的 route 值保持 `"反差" | "动作" | "唱歌"`
- "使用AI_movie_CY"部分不变
- 需要说明：兜底走剧本时，生图策略和编辑文本处理逻辑与原剧本路由一致
