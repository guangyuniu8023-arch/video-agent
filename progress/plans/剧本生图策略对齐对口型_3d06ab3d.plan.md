## 修改点1：生图策略伪代码（第45-64行）

**当前**：有 `group_photo` 分支、`图片数 != 分镜数` 判断、`for each 分镜` 循环

**改为**（与对口型-跟拍第15-24行一致）：

```
if 用户明确说"不生图/直接用我的图":
    所有分镜用 image_mapping

elif 单张图片是否多主体 == true and 无编辑需求:
    用 image_mapping（所有图都送给视频工具）

else:
    参考生图工具定义，选择合适的生图工具
```

## 修改点2：新增图片分配逻辑（对口型-跟拍第102-117行）

当前剧本-跟拍没有独立的图片分配表。需要在生图策略后面新增一节，与对口型-跟拍一致：

```
## 图片分配逻辑

无编辑需求时，不根据分镜数判断，看图片数、视频人数和图片主体数：

| 图片数 | 视频人数 | 图片主体数 | 生图策略 |
| 1张 | 单人 | 单人 | 生成1张（ID2i） |
| 1张 | 单人 | 多人 | 不生图（image_mapping） |
| 1张 | 多人 | 任意 | 不生图（image_mapping） |
| 多张 | 任意 | 任意 | 不生图（image_mapping） |

**特殊情况**：用户明确要求融合 → AI_create
**注意**：有编辑需求时，分发逻辑不变。
```

## 修改点3：工具表去掉 group_photo（第128-134行）

| `group_photo` | 合照 | images, ref_image, prompt |  ← 删除
